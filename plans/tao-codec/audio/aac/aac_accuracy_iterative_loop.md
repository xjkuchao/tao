# AAC 解码器精度迭代循环计划 (Loop v1)

## 1. 背景与目标

- 背景: 当前 AAC 解码器在 `data/1.m4a` 与 `data/2.m4a` 上已接近高精度, 但仍存在与参考实现的行为差异, 且 114 条覆盖样本中仍有失败和低精度记录.
- 总目标: 通过固定的 "对比 -> 修复 -> 验证 -> 提交/回滚" 循环, 持续收敛 AAC 解码精度与稳定性.
- 参考优先级:
  - 第一优先级: Fraunhofer FDK AAC.
  - 第二优先级: FFmpeg.
- 样本优先级:
  - P0: `data/1.m4a`, `data/2.m4a`.
  - P1: `plans/tao-codec/audio/aac/coverage/report.md` 中 114 条目标样本.

## 2. 功能大块划分与子功能拆分

### 功能点 1: 语法元素与位流同步 (最高优先级)

- 1-1: `ASC/ADTS/raw_data_block` 入口一致性与边界保护.
- 1-2: `SCE/CPE/LFE/PCE/FIL` 解析同步与跳过策略.
- 1-3: `common_window/ms_mask` 位数与组边界一致性.

### 功能点 2: ICS 语法正确性

- 2-1: `ics_info` 的 `window_sequence/max_sfb/grouping` 口径.
- 2-2: `section_data/scalefactor_data` 与 band type 映射.
- 2-3: `pulse/tns/gain_control` 条件分支与跳过语义.

### 功能点 3: 频谱重建与工具链

- 3-1: Huffman 解码与反量化.
- 3-2: MS Stereo 与 Intensity Stereo 应用顺序.
- 3-3: PNS 与 TNS 的频带范围及应用上限.

### 功能点 4: 时域合成路径

- 4-1: IMDCT 精度与舍入路径.
- 4-2: long/start/stop/short 窗切换与 overlap-add.
- 4-3: 首包裁剪, 尾包排空, 样本数对齐.

### 功能点 5: 声道布局与输出一致性

- 5-1: 默认声道映射与 PCE 显式布局.
- 5-2: 多声道 interleave 顺序.
- 5-3: `pts/duration/nb_samples` 与参考实现一致性.

### 功能点 6: 鲁棒性与能力边界

- 6-1: 非 AAC-LC profile 的结构化错误信息.
- 6-2: 损坏流容错路径 (不 panic, 可定位).
- 6-3: 封装层异常样本的稳定失败口径.

## 3. 固定循环流程 (每个子功能必须执行 1-6)

1. 建立子功能基线: 先对比 `data/1.m4a` 与 `data/2.m4a`.
2. 参考对比: 先 FDK, 后 FFmpeg, 再与 Tao 实现逐点比对.
3. 实施修复: 一次仅修改一个子功能闭环.
4. 快速验证: P0 两样本 + P1 失败/低精度抽样.
5. 判定处理:
   - 有效修复: 提交并记录证据.
   - 无效修复: 回滚并记录原因.
6. 切换下一个子功能并重复 1-5.

全子功能遍历后:

- 若整体仍未达标, 重新拆分子功能进入下一轮.

## 4. 有效修复判定规则

### 4.1 逻辑正确性优先

- 若能稳定证明 Tao 实现偏离 FDK 或 AAC 规范, 视为有效修复.
- 即使短期精度未显著上升, 只要逻辑证据成立, 仍允许提交.

### 4.2 精度变化辅助判定

- 一般情况下, 精度明显提升判定为有效.
- 若精度下降且缺少逻辑证据, 判定为无效并回滚.

### 4.3 联动修复判定

- 若当前修复依赖后续联动点才能显效, 可先判定有效.
- 提交和日志中必须标注联动依赖子功能.

## 5. 对比与验证口径

### 5.1 对比对象

- 规范: ISO/IEC 14496-3 AAC LC 相关章节.
- 参考实现:
  - FDK AAC 解码链路.
  - FFmpeg `libavcodec/aac*`.

### 5.2 测试样本优先级

- P0: `data/1.m4a`, `data/2.m4a` (每轮必测).
- P1: `coverage/report.md` 的 114 条样本 (阶段性回归).

### 5.3 最小验证集

- 快速门禁: 2 个 P0 样本.
- 短门禁: P1 的失败样本 + 精度<100%样本.
- 全量门禁: 114 条全量回归.

## 6. 分步任务与预期产出

1. 建立迭代基础设施.
   - 产出: 本计划, 基线文件, round log 文件.
2. 下载 114 条样本到 `data/aac_samples`.
   - 产出: `targets_local_index.md/json`.
3. 改造批测脚本支持本地优先.
   - 产出: `run_decoder.py` 新增 `--prefer-local-data` 等参数.
4. 执行功能点 1->6 循环.
   - 产出: 每子功能的证据记录, 修复提交或回滚记录.
5. 每轮结束更新基线与总评估.
   - 产出: `aac_accuracy_baseline.md`, `aac_accuracy_round_log.md`.

## 7. 依赖与前置条件

- 工具: `cargo`, `python3`, `curl`, `ffmpeg`, `ffprobe`.
- 数据: `data/1.m4a`, `data/2.m4a` 已就绪.
- 网络: 可访问 `https://samples.ffmpeg.org/`.

## 8. 验收标准

- P0 验收:
  - `data/1.m4a`, `data/2.m4a` 样本数差异为 0.
  - 精度达到或稳定维持 `>=99.99%`.
- P1 验收:
  - 114 条样本均可复现真实状态.
  - 相对当前基线, 成功率或精度分布单调改善.
- 稳定性验收:
  - 无新增 panic.
  - 错误信息中文且包含必要上下文.
