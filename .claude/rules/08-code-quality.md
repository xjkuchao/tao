# 代码质量要求

## 零警告容忍

- **不允许存在任何编译错误或 Clippy 警告**
- 未使用的 `use` 导入必须删除
- 未使用的变量, 函数, 类型定义必须删除或添加 `_` 前缀
- 禁止使用 `#[allow(...)]` 绕过 Clippy 检查, 除非有充分理由并添加详细注释

## 代码规范

- 避免硬编码的魔法数字或字符串, 使用常量或配置项
- 避免重复代码, 提取公共逻辑到工具函数或 trait
- 单个函数不超过 50 行, 复杂逻辑拆分为小函数

## 代码审查清单

每次修改后自我审查:

- 没有未使用的 imports, variables, functions
- 没有重复代码
- 没有硬编码的魔法数字或字符串
- 遵循项目代码风格
- 所有错误都有适当的处理
- 所有公开 API 都有文档注释
- 复杂逻辑有适当的注释说明
